# 単語プリント — 仕様・設計

A4用紙に印刷し、切って蛇腹（アコーディオン）に折るだけで単語帳ができる、印刷用レイアウト生成Webアプリ。

参考：Instagram reel（横に折る → 縦に切る → 蛇腹の単語帳）
https://www.instagram.com/reel/C4InnXNJS7W/

## 1. 概要

- ユーザーが単語（表面）と訳（裏面）のペアを入力すると、A4印刷用のレイアウトを生成する。
- 印刷 → 縦に切って帯に分ける → 各帯を蛇腹に折る、で単語帳が完成する。
- 片面印刷のみ。両面印刷や用紙の回転は不要。
- ログイン不要・DB不要。その場で入力して印刷する使い捨てツール。

## 2. 紙の作り方（プロダクトが前提とする手順）

参考reelの方式に合わせる：

1. **横に折る**：単語が書けるくらいの高さで、紙を上から下へアコーディオン状に折る。
2. **縦に切る**：折った状態のまま、縦に N 等分して切り分ける（reelでは4等分）。これで N 本の帯（縦長のアコーディオン）ができる。
3. 各帯を蛇腹に折りたたむと、パタパタと1コマずつめくれる単語帳になる。手前を向く面に単語（表面）、奥を向く面に訳（裏面）が交互に現れる。

### レイアウトの構造

```
1ページ = N本の縦帯（左右に並ぶ・切り離し用）
  各帯 = 上から下へ積まれた複数パネル（折り目で区切られる）
    パネルは上から順に：表面 → 裏面 → 表面 → 裏面 …
```

- **切り取り線（縦）**：ページを左右に N 分割して帯を切り離す線。
- **折り線（横）**：各帯の中で、単語が書けるくらいの高さ（パネル高さ）ごとに折る位置。

## 3. 印刷レイアウト仕様

- 用紙：A4縦。
- **用紙マージン：0（フチなし）**。帯を紙の端いっぱいまで使う（`@page { size: A4; margin: 0; }`）。
  - 注：家庭用プリンタには物理的な印字不可領域があるため、実際にはプリンタ設定の「フチなし印刷」に依存する。フチなし非対応環境向けに、極小マージンへのフォールバックを検討。
- **黒一色・最小限のインク**で印刷する。
- **罫線は紙全面には引かない**。折る位置・切る位置には端に小さな目印（トンボ状のティック）だけを印字してインクを節約する。ユーザーは定規を目印同士に合わせて折る／切る。
  - 折りの目印：各帯の左右の端に、パネル境界ごとの短いティック。
  - 切りの目印：帯の境界の上下端に短いティック（＋ハサミアイコン等）。
- **面ラベル（表/裏）や枠線は印刷しない**。印字されるのは単語・訳の文字と端の目印のみ。
- パネル高さ・帯の本数はいずれも可変（設定で変更）。
- 各印刷ページに**ページ番号（例：P. 1 / 3）**を印字し、どこで紙が分かれるか分かるようにする。

### ページ／帯をまたぐときのルール

- 単語（表面）と訳（裏面）のペアは必ず**同じ帯**に収める。帯の残りパネルがペア分（2パネル）に満たない場合は、そのペアごと次の帯の先頭へ送る。
- 帯も埋まったら次ページへ送る。**ペアの途中で帯や紙が切り替わることはない**。
- 収容量の計算：
  - 1帯あたりのパネル数 = floor(ページ有効高さ ÷ パネル高さ)
  - 1ページの帯数 = 設定値 N
  - 1ページの容量（パネル数）= 帯数 × 1帯あたりパネル数

## 4. Web版（入力・編集画面）

- **入力欄は「表面」「裏面」の2列テーブル**。
  - スプレッドシートで2列を選択してコピーし、テーブル左上セルに貼り付け（TSV貼り付け）で一括入力できる。
  - ヘッダーは「表面」「裏面」のみ（「単語」「訳」等の補足語は付けない）。
- **ページの境目は太い区切り線で表示**（区切り専用の行は使わない）。1ページ目に収まる最後のペアの下に太線を引き、「ここまでが1ページ目」と分かるようにする。現在のレイアウト設定（帯数・パネル高さ）から自動計算する。
- レイアウト設定：帯の本数、パネル高さ（1語あたりの折り幅）、フォントサイズ等を調整できる。
- **プレビュー**：A4を縮小表示し、印刷結果（切り／折りの目印・ページ番号を含む）をそのまま確認できる。画面と印刷で同じCSS（mm単位）を共有しWYSIWYGにする。
- **印刷**：`window.print()` を呼ぶだけ。PDF化はブラウザの印刷ダイアログに任せる（サーバー側PDF生成はしない）。複数ページを1回の印刷操作で出力できる。

### リデザイン（2026-07）

- 画面プレビューは廃止した。編集画面には印刷結果の縮小表示を出さず、代わりに「◯語 ・ ◯ページ」という語数・ページ数のサマリ行のみを表示する。印刷用のDOM（`.sheet`）自体はSSR/CSR問わず常に生成しており、`@media print` のときだけ表示される（画面表示時は `.print-sheets { display: none }`）。
- レイアウト設定（帯の本数・パネル高さ・フォントサイズ）はユーザー調整UIを廃止し、`DEFAULTS`（帯4本・パネル高さ20mm・基準フォント14pt）に固定した。長い語がパネル幅に収まらない場合は、パネルごとに `src/lib/fit.ts` の `fitFontSizePt` でフォントサイズを自動的に縮小してフィットさせる（最小8pt）。
- UI言語はログイン不要のまま `Accept-Language` ヘッダーから日本語／英語を自動判定する（`src/lib/i18n.ts`）。SSR時に決定した言語がHTMLの `lang` 属性・タイトル・meta descriptionにも反映される。
- 印刷物そのものの仕様（§3：用紙・マージン・インク・目印・ページ番号など）は変更していない。

## 5. 入力データフォーマット

Web版のテーブル入力に加え、テキスト貼り付けでも以下を受け付ける（自動判定）：

1. **区切り文字あり（推奨）**：1行1組、カンマまたはタブ区切り。
   ```
   Apple,りんご
   Banana,ばなな
   ```
   （スプレッドシートからのコピーはタブ区切りTSVになる）
2. **交互1行ずつ**：区切り文字が無ければ、奇数行＝表面、偶数行＝裏面として解釈。
   ```
   Apple
   りんご
   Banana
   ばなな
   ```

- 空行は無視、前後の空白はtrim。
- 交互形式でペアが作れない（行数が奇数）場合はエラー表示。

## 6. 技術構成

- **Cloudflare Workers + Hono**：ルーティングとSSR。保存・DBが不要なため、実質 `GET /` でHTMLシェルを返す薄い構成。
- **BarefootJS**：入力欄・レイアウト設定・プレビューの反応的な部分（island）を担当。TSXで書き、コンパイル時にテンプレート＋最小限のハイドレーションJSを生成する。SSRしたHTMLに対し、入力→再計算→プレビュー更新の部分だけクライアントJSを当てる。
- パース・レイアウト計算・ページ分割はすべてクライアントサイドで実行（キー入力ごとの再計算でも軽量）。サーバー往復は不要。
- MVPでは状態保存なし。

## 7. 非機能要件

- ログイン不要・DB不要。
- 依存を増やさずWorkers上で軽量に動作。
- 編集はPC想定だが、スマホでも極端に崩れない。
- 印刷CSSと画面プレビューCSSを共通化しWYSIWYGを保つ。
